    Declaring & Assigning

    • Declare variables before Assigning
    • After ES6, JS uses const & let to declare
    • const = variable will not be reassigned to another value
    • let = variable can be reassigned in program
    • to see value of varible, enter name of variable
    • can declare and assign multiple variables at same time
    • If variable references non-primitive datatype such as array, function or object then CONST will not make it immutable
    • Using cONST prevents you from reassigning a variable to another object
    • Try to use CONST - makes proj more predictable. Helps avoid bugs

    Scope

    • CONST & LET = block scoped, value exists inside block declared in
    • Scope = Where a constant or variable is accessible by the program
    • 2 common scopes - Global and Local

    Global Scope

    • Any variable declared out of block
    • Accessible anywhere in program
    • Not considered good practice, as variables with same name clash & overwrite each other

    Local Scope

    • Blocks can be used to create local Scope
    • Any variables defined inside a block using LET or CONST will only be avail inside that block
    • If LET & CONST are not used, the variable will be available out side the block
    • e.g. let b=2; {b=4; b;} returns 4 (check outside block) b; returns 4;
    • Good pracetice to use LET & CONST to declare variables to ensure they are block scoped.

    Naming Constants & Variables

    • Give constants & variables sensible names that describe what the variables represents
    • Can start with: upper case, lower case, underscore _, dollar sign $.
    • Can not start with numbers, but CAN contain them.

    Variable Naming

    • Variables that Start with _ (underscore) refer to private properties and methods
    • $ sign is used by the query library. It is proper practice not to use as can cause problems
    • They are case sensitive.
    • When using multiple words try to use Camel Case. this is the best practice.

    Direct Assignment and Assignment by Reference

    • When assign primitive value to variable, any changes you make are made directly to that value.
    • If you assign non-primitive value to variable, this is done by reference So changes made after will affect all references to that object.

    Method

    Numbers

    • Can be integers,
      • floating point numbers
    • isInteger() method can help distinguish between types of numbers by identifying if the number is an integer
    • Numbers also have constructor function
    • new Number(3) returns [Number 3]
    • Preference is for number literals though

    Octal/Hexadecimal Numbers

    • If the numbers starts with 0x it is considered to be in hexidec (base16) notation
    • e.g. 0xAF //A represents 10, F represents 15 so returns 175
    • If starts with 0o considered to be Octal (base 8) notation
    • e.g. 0o47 // 4 eights and 7 units returns 39
    • ES6 supports binary literals
    • If number starts with zero followed by letter b consider to be binary (base 2) notation
    • 0b010 //1 eight, 0 fours, 1 two and 0 units returns 10

    Exponential Notation

    • Numbers also represented in exponential notation = shorthand for "Multiply by 10 to power of"
    • also called "scientific notation
    • e.g. 1e6; // 1 multiplied by 10 to the power 6 returns 1,000,000
    • e.g. 2E3; // 2 multipled by 10^3
    • Decimal values created by using negative index values:
    • 2.5e-3; //25 multiplied by 10 to power of -3 (.001) returns 0.0025

    Number Methods

    • Also have built in methods
    • Be careful with dot notation and not to confuse it with the decimal point
    • eg. 5..toExponential();
    • Can also put a space between e.g 5 .toExponential();
    • Or write as a decimal e.g. 5.0.toExponential();
